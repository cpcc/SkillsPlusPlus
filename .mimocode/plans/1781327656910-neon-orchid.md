# Plan: 引入 Playwright 进行端到端测试

## 目标
为 skills++ Tauri 应用引入 Playwright E2E 测试框架，覆盖主要页面的用户交互场景。

## 技术方案

### 1. 安装依赖
在 `apps/desktop` 目录安装 Playwright：
```bash
pnpm add -D @playwright/test
pnpm exec playwright install chromium
```

### 2. 创建配置文件
创建 `apps/desktop/playwright.config.ts`：
- 指向 Vite dev server（`http://localhost:1420`）
- 配置 Chromium 浏览器
- 设置测试目录 `e2e/`
- 配置 webServer 自动启动 Vite

### 3. 创建测试目录结构
```
apps/desktop/e2e/
├── discover.spec.ts    # 发现页测试
├── installed.spec.ts   # 已安装页测试
├── tools.spec.ts       # 工具页测试
├── settings.spec.ts    # 设置页测试
└── navigation.spec.ts  # 导航测试
```

### 4. 测试用例设计

#### navigation.spec.ts
- 验证默认重定向到 /discover
- 验证侧边栏导航切换

#### discover.spec.ts
- 验证页面标题"发现"
- 验证搜索框存在
- 验证刷新按钮功能
- 验证 SkillCard 渲染

#### installed.spec.ts
- 验证页面标题"已安装"
- 验证空状态提示
- 验证刷新状态按钮

#### tools.spec.ts
- 验证页面标题"工具与目录"
- 验证新增目录按钮
- 验证重新扫描按钮

#### settings.spec.ts
- 验证页面标题"设置"
- 验证应用信息显示

### 5. 添加 npm scripts
在 `apps/desktop/package.json` 添加：
```json
"test:e2e": "playwright test",
"test:e2e:ui": "playwright test --ui"
```

## 关键文件
- `apps/desktop/package.json` - 添加依赖和 scripts
- `apps/desktop/playwright.config.ts` - 新建 Playwright 配置
- `apps/desktop/e2e/*.spec.ts` - 新建测试文件

## 验证方式
```bash
cd apps/desktop
pnpm test:e2e
```
