# 安装策略（Install Strategy）

skills++ 支持四种安装策略，让用户根据场景灵活选择 skill 的落地方式。每个来源适配器（adapter）声明默认策略，用户可在安装对话框中覆盖。

---

## 四种策略一览

| 策略 | 枚举值 | 说明 | 适用场景 |
|------|--------|------|----------|
| Git Clone | `git` | `git clone --depth=1` 浅克隆到目标目录 | GitHub 仓库，需要后续 `git pull` 更新 |
| 拷贝 | `copy` | 下载 tar.gz/zip → 解压 → strip 顶层目录 → 写入目标，去掉 `.git` | 不依赖 git、离线分发 |
| 压缩包 | `archive` | 与 copy 底层相同（HTTP GET + 解压），语义上特指 zip 来源 | LobeHub 等 zip 分发站 |
| 软链 + 规范存储 | `skills_cli` | 内容装到 `~/.agents/skills/<name>/`，在 AI 工具目录创建 symlink，写 lockfile | 与 vercel-labs `npx skills` 完全互通 |

> 枚举定义：`apps/desktop/src-tauri/src/models/mod.rs` → `InstallStrategy`（serde snake_case 序列化）

## 策略 1：Git Clone (`git`)

保留原有行为，无变化。

```rust
// services/install.rs:129
pub fn git_clone(repo_url: &str, target: &Path) -> Result<Vec<String>, String>
```

- `git clone --depth=1 --progress <repo_url> <target>`
- 更新检测：`git fetch` + `git rev-list` 比对本地与远端 HEAD

## 策略 2 & 3：拷贝 / 压缩包 (`copy` / `archive`)

两者底层共享同一套下载 + 解压逻辑：

```rust
// services/install.rs:164
fn download_and_extract(url: &str, target: &Path, strip_git: bool) -> Result<(), String>
```

### 流程

1. **HTTP GET** 下载归档字节（`blocking_get_bytes`，在独立线程跑 tokio runtime）
2. **Magic bytes 格式检测**：
   - `1f 8b` → gzip/tar.gz（`extract_tar_gz`）
   - `50 4b 03 04` → zip（`extract_zip`）
   - 默认按 tar.gz 尝试，失败报错
3. **Strip 顶层目录**：若解压后只有一个子目录（如 `repo-main/`），取它作为内容根
4. **去掉 `.git`**：`move_contents` 时跳过 `.git` 目录
5. **强制校验 SKILL.md**：装完目录必须有 `SKILL.md`（大小写不敏感：`SKILL.md` / `skill.md` / `Skill.md`），否则删除半成品目录并报错

### archive_url 来源

GitHub adapter 自动合成 codeload 地址：

```rust
// services/adapters/github.rs
pub fn github_archive_url(owner: &str, repo: &str) -> String {
    format!("https://codeload.github.com/{owner}/{repo}/tar.gz/refs/heads/main")
}
```

### 更新检测

重新下载到临时目录，计算 SHA1 目录哈希，与 `installed_skills.content_hash` 列比对：

```rust
// services/install.rs:757
InstallStrategy::Copy | InstallStrategy::Archive => {
    if let Some(stored) = s.content_hash.as_deref() {
        match hash_differs_for(&url, stored) {
            Ok(true) => new_status = "update_available",
            ...
        }
    }
}
```

## 策略 4：软链 + 规范存储 (`skills_cli`)

完整复刻 vercel-labs [`npx skills`](https://github.com/vercel-labs) 的安装范式，实现与本机已有 `npx skills` 安装**互通**。

### 目录布局

```
~/.agents/
├── .skill-lock.json          ← 全局 lockfile（version: 3）
└── skills/
    └── <skill-name>/         ← canonical store（真实文件）
        ├── SKILL.md
        └── ...

~/.claude/skills/
    └── <skill-name>          ← symlink → ~/.agents/skills/<skill-name>/
```

### 安装流程（`install_skills_cli`）

```
services/install.rs:331

1. canonical = ~/.agents/skills/<sanitize(name)>/
2. 若 canonical 不存在或为空：
   ├─ 有 archive_url → download_and_extract（tar.gz）
   └─ 无 archive_url → git_clone + 删 .git
3. 校验 SKILL.md 存在（不存在 → 删 canonical + 报错）
4. 解析 SKILL.md frontmatter，按 manifest.name 规范化目录名
5. 在 agent_link_dir/<name> 创建 symlink → canonical
6. 计算 canonical 目录 SHA1 哈希
7. 写 lockfile 条目（upsert）
8. 返回 { canonical_path, symlink_path, content_hash }
```

### Lockfile 格式（`~/.agents/.skill-lock.json`）

```json
{
  "version": 3,
  "skills": {
    "my-skill": {
      "source": "owner/repo",
      "sourceType": "github",
      "sourceUrl": "https://github.com/owner/repo",
      "skillPath": "SKILL.md",
      "skillFolderHash": "a1b2c3...",
      "installedAt": "2026-06-13T12:00:00.000Z",
      "updatedAt": "2026-06-13T12:00:00.000Z"
    }
  }
}
```

字段与 vercel-labs `npx skills` 完全一致，双向互通。

### 更新检测

重算 canonical 目录哈希，与 lockfile 里的 `skillFolderHash` 比对；不一致 → `changed`。

### 卸载行为

`skills_cli` 策略卸载时**只删 symlink**，保留 canonical 目录（因为 canonical 可能被多个 agent 目录引用）。canonical 清理留给后续迭代的「清理规范存储」功能。

---

## SKILL.md 校验（所有策略通用）

**强制要求**：任何策略装完后，目标目录必须有 `SKILL.md`（大小写不敏感）。没有则视为安装失败，半成品目录会被删除。

```rust
// services/install.rs:303
fn has_skill_md(dir: &Path) -> bool {
    for name in ["SKILL.md", "skill.md", "Skill.md"] {
        if dir.join(name).exists() { return true; }
    }
    false
}
```

### Frontmatter 解析（`services/skill_md.rs`）

零依赖最小 YAML 子集解析器：
- 读取首对 `---` 之间的内容
- 按行 `key: value` 提取 `name` / `description`
- `sanitize_name()`：非 `[a-zA-Z0-9-_.]` 字符替换为 `-`
- `normalize_skill_dir()`：若目录名与 manifest.name 不一致则 rename

---

## 目录哈希（`compute_folder_hash`）

```rust
// services/canonical_store.rs
pub fn compute_folder_hash(path: &Path) -> String
```

- 递归遍历所有文件（跳过 `.git` 目录）
- 对每个文件的「相对路径 + 文件内容」做 SHA1
- 返回十六进制字符串
- **跳过 `.git` 保证哈希稳定**（git 元数据不影响内容指纹）

## Adapter 默认策略

每个 adapter 通过 `default_install_strategy()` 声明默认策略：

| Adapter | 默认策略 | 备注 |
|---------|----------|------|
| `GithubAdapter` | `git` | 额外填充 `archive_url`（codeload tar.gz） |
| `LobehubAdapter` | `archive` | zip 分发 |
| `StubAdapter` | 可配置 | source_registry 中 skillhub/clawhub → `copy`，skillsmp → `archive` |

用户在 InstallDialog 可覆盖默认值。
