# skills++ Roadmap

## 1. Roadmap 目标

本文档基于《开发计划》提炼出面向执行与对齐的产品研发路线图，用于回答三个问题：

1. **先做什么**：明确 MVP 的阶段顺序与依赖关系。
2. **何时交付什么**：定义每个阶段的核心产出与里程碑。
3. **如何判断可进入下一阶段**：给出阶段验收口径与关键风险。

当前 roadmap 聚焦 **MVP 10 周交付**，默认首版覆盖：

1. 5 个 skill 来源站聚合
2. 10 个 AI 工具目录识别与安装支持
3. skill 安装、卸载、重装与日志
4. 已安装 skill 管理
5. macOS / Windows / Linux 三端可用

## 3. 阶段路线图

## P0：调研与技术验证

### 阶段目标
把“能不能做”变成“怎么做”，形成后续开发的输入基线。

### 本阶段重点
1. 调研 10 个 AI 工具的默认 skill 目录与平台差异。
2. 调研 5 个来源站的抓取方式、字段结构与限制条件。
3. 验证 `skills` CLI 的目录探测与安装行为。
4. 明确统一数据模型、安装策略与错误场景。

### 关键产出
1. 工具目录规则清单
2. 来源站适配清单
3. 安装策略矩阵
4. 错误码与用户提示基线

### Checkpoints
- [ ] 完成 10 个 AI 工具目录规则调研
- [ ] 完成 5 个来源站数据接入调研
- [ ] 完成 `skills` CLI 行为验证
- [ ] 完成统一字段映射与空值兜底策略
- [ ] 完成至少 1 条安装样例链路验证

### 进入下一阶段条件
1. 至少 10 个工具具备默认目录或可行兜底方案。
2. 至少 5 个来源站具备可实现的数据接入方案。
3. 至少 1 条完整安装样例链路跑通。

## P1：基础框架与数据底座

### 阶段目标
搭建可持续开发的桌面应用底座，先打通前后端、数据库和页面框架。

### 本阶段重点
1. 初始化 Tauri + React + TypeScript + Tailwind 项目。
2. 搭建路由、布局、导航、通知和弹窗能力。
3. 建立 SQLite 表结构、迁移和 Repository 层。
4. 建立 Tauri Command / Service / Adapter / Repository 分层。
5. 搭建发现、已安装、工具与目录、设置四个一级页面骨架。

### 关键产出
1. 可启动的桌面应用壳
2. 可读写的本地数据库
3. 稳定的 IPC 通信接口
4. 前端页面骨架与基础组件库

### Checkpoints
- [ ] 完成 Tauri + React + TypeScript 项目初始化
- [ ] 完成路由、布局、导航、通知、弹窗基础设施
- [ ] 完成 SQLite 表结构与迁移机制
- [ ] 完成共享 schema 与错误码体系
- [ ] 完成 Tauri Command / Service / Adapter / Repository 分层
- [ ] 完成发现、已安装、工具与目录、设置页面空态

### 进入下一阶段条件
1. 本地数据库初始化成功。
2. 前后端基础命令调用成功。
3. 四个主页面可以稳定访问。

---

2. 实现目录存在性、可写性和 skill 数量扫描。
3. 实现工具与目录页展示与编辑能力。
4. 实现手动新增目录、禁用目录、设置默认目录。
5. 实现首次启动“扫描 → 结果展示 → 用户补充配置”引导流程。

### 关键产出
1. 自动识别目录列表
2. 自定义目录配置能力
3. 默认安装目录推荐能力
4. 首次启动初始化引导

### Checkpoints
- [ ] 完成工具目录规则模型与跨平台路径展开
- [ ] 完成目录存在性、可写性、skill 数量扫描
- [ ] 完成工具与目录页展示
- [ ] 完成手动新增目录、禁用目录、默认目录设置
- [ ] 完成首次启动扫描与引导流程
- [ ] 完成 `skills` CLI 目录探测辅助接入

### 进入下一阶段条件
1. 用户首次进入可看到扫描结果。
2. 用户可保存目录配置并在重启后保留。
3. 系统可为安装流程提供默认目录输入。

## P3：来源聚合与发现体验

### 阶段目标
把分散的 skill 来源统一成可搜索、可筛选、可查看详情的发现入口。

### 本阶段重点
1. 接入 5 个来源站适配器。
2. 建立统一 SkillItem 结构与字段兜底规则。
3. 实现本地缓存与刷新策略。
4. 实现发现页：搜索、来源筛选、标签筛选、工具筛选、排序。
5. 实现 skill 详情页。
6. 实现来源站启用/禁用配置。

### 关键产出
1. 来源聚合列表
2. 发现页完整检索能力
3. skill 详情展示
4. 来源站可配置开关

### Checkpoints
- [ ] 完成 5 个来源站适配器接入
- [ ] 完成 SkillItem 标准化与兼容工具推断
- [ ] 完成本地缓存与刷新策略
- [ ] 完成发现页搜索、筛选、排序
- [ ] 完成 skill 详情页
- [ ] 完成来源站启用/禁用设置

### 进入下一阶段条件
1. 用户可浏览 5 个来源站聚合数据。
2. 搜索与筛选结果正确。
3. 详情页可展示来源、仓库与兼容信息。

## 7. 风险观察点

### 高风险项
1. 来源站结构变化导致抓取失效
2. AI 工具目录规则变更导致扫描不准
3. 外部 CLI 行为变化导致辅助探测不稳定
4. 三平台文件系统与权限差异导致安装失败

### 对应策略
1. 采用 Adapter 隔离来源站差异，并支持来源站单独禁用。
2. 目录规则配置化，并允许用户手动补充与覆盖。
3. 外部 CLI 仅作为辅助，不作为唯一安装依赖。
4. 在 P6 前完成平台专项回归清单。

## 9. Roadmap 完成定义

当以下条件全部满足时，可认为本 roadmap 对应的 MVP 阶段完成：

1. 用户能在应用内完成 **发现 → 安装 → 管理 → 卸载** 全链路。
2. 5 个来源站、10 个工具目录支持达到 PRD 的 MVP 下限。
3. 三平台核心流程验证通过。
4. 关键异常场景具备用户提示与本地日志。
5. 项目具备进入下一轮增强迭代的技术底座。

- [ ] 发现 → 安装 → 管理 → 卸载全链路可用
- [ ] 5 个来源站支持达标
- [ ] 10 个工具目录支持达标
- [ ] 三平台核心流程验证通过
- [ ] 关键异常场景具备提示与日志
